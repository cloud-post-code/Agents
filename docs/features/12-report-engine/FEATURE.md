# Feature: Report Engine

## What to Build

The report generation infrastructure: Celery pipeline, WeasyPrint renderer, storage layer, and report archive UI. No report templates yet (those are features 13–15).

### Backend — Core Infrastructure

- `generate_report` shared tool implementation: validates params, enqueues Celery task, creates pending report row, returns `{report_id, estimated_seconds}`
- `POST /api/v1/reports/generate` — HTTP trigger (for scheduled reports via Celery Beat)
- `GET /api/v1/reports` — paginated list; filterable by `generated_by` (agent role), `format`, `date_range`
- `GET /api/v1/reports/{id}` — report detail + download URL
- `GET /api/v1/reports/{id}/download` — redirect to presigned URL or serve from volume

### Celery Report Pipeline

```python
@celery_app.task(bind=True, max_retries=2)
def generate_report_task(self, report_id: str, template_id: str, params: dict, tenant_id: str):
    1. Set app.tenant_id
    2. Fetch data via template-specific data pipeline
    3. Run LLM synthesis pass (summary paragraph for report header)
    4. Render Jinja2 HTML template with data + synthesis
    5. WeasyPrint: HTML → PDF bytes
    6. Upload to storage (Volume in dev, R2 in prod)
    7. Update reports row: storage_url, size_bytes, status='complete'
    8. Publish to Redis: {type: "report_ready", report_id, title, download_url}
    9. Write notification row
```

### Jinja2 Base Template

All reports share a base HTML template with:
- Artisan brand header (business name, logo from tenant settings, report title, date range)
- Dynamic `{% block content %}` for each report template
- Print-optimized CSS: `@page` rule, page break controls, no viewport units
- Report footer: "Generated by [Agent Name] on [Date]"

### Celery Beat Schedules

Registered schedules for all 18 report templates per their cadence. Schedules run for all tenants; each task scopes to one tenant via `tenant_id`.

### Frontend — Report Archive

- `/reports` page: list of all generated reports, newest first
- Each report card: title, agent badge, format badge (PDF/HTML), generated date, download button
- Filter by agent, date range
- Download button fetches presigned URL, triggers browser download
- Empty state: "Your agents will generate reports here automatically"
- Report notification links navigate to the correct report card

## Out of Scope

- Individual report data pipelines (features 13–15)
- Custom report builder
- Report scheduling UI (schedules are fixed in Celery Beat config)
